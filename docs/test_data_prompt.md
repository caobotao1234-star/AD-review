# 测试数据生成提示词

## 一、生成商品 JSON 文件

将以下提示词发给 GPT，生成 10 个测试用的商品 JSON 文件：

---

**提示词：**

```
请帮我生成 10 个抖音商品页的结构化数据 JSON 文件，用于测试广告合规审查系统。

每个 JSON 文件的格式如下：
{
  "user_id": "用户ID（如 user_001）",
  "title": "商品标题",
  "category": "商品类目",
  "price": "当前价格",
  "original_price": "原价（可选）",
  "efficacy_claims": ["功效声明列表"],
  "specifications": "规格描述",
  "origin": "产地",
  "certifications": ["资质/认证列表"]
}

请生成以下 10 个案例（文件名用 case_01 到 case_10）：

1. case_01: 保健食品，正常合规，无违规
2. case_02: 保健食品，使用"根治""治愈"等医疗暗示词汇
3. case_03: 护肤品，使用"全网最低价""史上最强"等绝对化用语
4. case_04: 医疗器械，声称"100%有效""无副作用"
5. case_05: 金融理财产品，声称"保本保息""零风险"
6. case_06: 教育培训，声称"包过""100%通过率"
7. case_07: 食品，图片标价 29.9 但 JSON 中 price 为 59.9（价格矛盾）
8. case_08: 服装，图片写"纯棉"但 JSON specifications 为"聚酯纤维"（材质矛盾）
9. case_09: 电子产品，使用"央视推荐""国务院认证"等伪造资质
10. case_10: 日用品，轻微擦边（如"效果超好""强烈推荐"），属于灰色地带

每个 JSON 单独输出，标注文件名。
```

---

## 二、生成广告图片

将以下提示词发给 GPT（使用 image generation / DALL-E），为每个案例生成对应的广告图片：

---

**提示词模板（逐个生成）：**

```
请生成一张抖音风格的商品广告图片。

要求：
- 尺寸：竖版 9:16 比例
- 风格：抖音电商直播间截图风格，有醒目的促销文案
- 必须包含中文文字

具体内容：
[根据每个 case 填写]
```

**各案例的具体内容：**

case_01（合规保健食品）：
```
商品：维生素C片
文案：「每日一片 补充活力」「天然提取 品质保障」
价格标签：¥89
风格：清新健康风，绿色调
```

case_02（医疗暗示）：
```
商品：某草本保健品
文案：「根治失眠 一盒见效」「祖传秘方 药到病除」
价格标签：¥199
风格：中药风格，深色背景，大字报式
```

case_03（绝对化用语）：
```
商品：美白精华液
文案：「全网最低价」「史上最强美白 第一品牌」
价格标签：¥128 原价¥399
风格：粉色系，闪光特效，大促风格
```

case_04（医疗器械虚假）：
```
商品：家用血压仪
文案：「100%精准测量」「无副作用 替代医院检查」「保证见效」
价格标签：¥299
风格：白色医疗风，有伪造的"FDA认证"标志
```

case_05（金融违规）：
```
商品：理财课程/产品
文案：「保本保息 年化收益30%」「零风险 稳赚不赔」
价格标签：¥9.9 限时
风格：金色系，财富感，有虚假收益截图
```

case_06（教育培训）：
```
商品：公务员考试培训
文案：「包过班 不过全额退款」「100%通过率 内部资料」
价格标签：¥4999
风格：红色系，紧迫感，倒计时元素
```

case_07（价格矛盾）：
```
商品：坚果零食大礼包
文案：「限时特惠 ¥29.9」「原价¥199 今日跳楼价」
价格标签：¥29.9（注意：JSON中实际价格是59.9，故意制造矛盾）
风格：黄色促销风，大字价格
```

case_08（材质矛盾）：
```
商品：T恤衫
文案：「100%纯棉 亲肤透气」「新疆长绒棉 高端面料」
价格标签：¥79
风格：简约白色背景，衣服展示图（注意：JSON中材质是聚酯纤维）
```

case_09（伪造资质）：
```
商品：空气净化器
文案：「央视推荐品牌」「国务院认证 驰名商标」「免检产品」
价格标签：¥1299
风格：有伪造的央视logo和红色印章
```

case_10（灰色地带）：
```
商品：洗衣液
文案：「效果超好 强烈推荐」「用过都说好」
价格标签：¥39.9
风格：日常家居风，普通促销
```

## 三、文件命名和放置

生成后按以下方式放置：

```
input/
├── case_01.png    (或 .jpg)
├── case_01.json
├── case_02.png
├── case_02.json
├── ...
├── case_10.png
└── case_10.json
```

图片和 JSON 文件名必须一致（只是后缀不同）。
